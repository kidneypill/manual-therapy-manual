기능

체계적인 지식으로

Concept 파일 하나로 질환의 전체 지식 체계를 구성합니다.

자동 섹션 동기화

Concepts 파일을 저장하면 Overview, Mechanism, Assessment, Treatment, Prognosis가 자동으로 생성·갱신됩니다.

UID 기반 연결 구조

모든 지식 노드는 고유 UID로 식별됩니다. 질환, 검사, 치료 간의 참조 관계가 명확히 추적됩니다.

학파별 독립 라이브러리

Applied Kinesiology, Orthopedic 등 학파별 지식이 독립된 섹션으로 분리되어 혼용을 방지합니다.

무결성 자동 검사

check_integrity.py가 UID 중복, 필수 필드 누락, 링크 오류를 자동으로 검출합니다.

YAML 프론트매터

각 파일은 구조화된 메타데이터를 YAML로 관리합니다. 기존 파일에 이식하거나 스키마를 확장하기 쉽습니다.

저장 시 즉시 실행

Claude Code hook이 파일 저장을 감지해 동기화를 자동으로 실행합니다. 별도 명령 없이 항상 최신 상태.

워크플로

한 번 작성,
자동으로 전개

Concept 파일을 작성하는 것만으로 전체 지식 구조가 완성됩니다.

01
Concept 작성

Concepts/Concept-{질환명}.md에 통합 원본을 작성합니다. 표준 헤더 구조를 따릅니다.

02
자동 감지

파일 저장 시 Claude Code hook이 변경을 감지하고 sync_concepts.py를 실행합니다.

03
섹션 분리

스크립트가 헤딩 구조를 파싱해 섹션별 파일을 Conditions/{slug}/에 생성합니다.

04
지식 완성

Overview, Mechanism, Assessment, Treatment, Prognosis가 각각 독립 파일로 완성됩니다.
